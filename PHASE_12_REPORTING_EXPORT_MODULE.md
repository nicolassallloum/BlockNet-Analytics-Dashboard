# Phase 12 — Reporting and Export Module

## 1. Phase Objective

Build the reporting and export module for dashboard data.

This phase allows analysts and admins to export blockchain metrics, alert history, market summaries, and API sync logs into CSV, Excel, and PDF formats.

---

## 2. Report Types

| Report Type | Description |
|-------------|-------------|
| `network_summary` | Summary for all active networks |
| `market_metrics` | Price, market cap, volume, supply |
| `defi_metrics` | TVL, protocol count, DEX volume |
| `stablecoin_metrics` | Stablecoin supply and transfers |
| `historical_metrics` | Time-series chart data |
| `alerts` | Alert history |
| `api_sync_logs` | Provider sync history |

---

## 4. Report Request Payload

Endpoint:

```text
POST /api/v1/reports/export
```

Payload:

```json
{
  "report_type": "market_metrics",
  "format": "xlsx",
  "network_key": "bitcoin",
  "date_from": "2026-01-01",
  "date_to": "2026-01-31",
  "filters": {
    "metric_group": "market",
    "metric_key": "price_usd"
  }
}
```

---

## 5. Validation Rules

Create request:

```bash
php artisan make:request Api/V1/ExportReportRequest
```

Rules:

```php
public function rules(): array
{
    return [
        'report_type' => ['required', 'string', 'in:network_summary,market_metrics,defi_metrics,stablecoin_metrics,historical_metrics,alerts,api_sync_logs'],
        'format' => ['required', 'string', 'in:csv,xlsx,pdf'],
        'network_key' => ['nullable', 'string', 'exists:blockchain_dashboard.blockchain_networks,network_key'],
        'date_from' => ['nullable', 'date'],
        'date_to' => ['nullable', 'date', 'after_or_equal:date_from'],
        'filters' => ['nullable', 'array'],
    ];
}
```

## 7. Export File Storage

Recommended path:

```text
storage/app/reports/
```

File naming standard:

```text
{report_type}_{YYYYMMDD_HHMMSS}_{user_id}.{format}
```

Example:

```text
market_metrics_20260101_120000_1.xlsx
```

---

## 8. CSV Export

For CSV exports:

- Use streamed response for small and medium reports.
- Use queued export for large reports.
- Include header row.
- Use UTF-8 encoding.

Required headers example for market metrics:

```text
Network, Symbol, Price USD, Market Cap USD, Volume 24h USD, 24h Change %, Recorded At
```

---

## 9. Excel Export

Create export class:

```bash
php artisan make:export MarketMetricsExport
```

Example:

```php
class MarketMetricsExport implements FromCollection, WithHeadings
{
    public function __construct(private Collection $rows) {}

    public function collection()
    {
        return $this->rows;
    }

    public function headings(): array
    {
        return [
            'Network',
            'Symbol',
            'Price USD',
            'Market Cap USD',
            'Volume 24h USD',
            '24h Change %',
            'Recorded At',
        ];
    }
}
```

## 11. Report Exports Table

Every report generation must create a row in `report_exports`.

Required fields:

- `user_id`
- `report_type`
- `file_format`
- `file_name`
- `file_path`
- `filters`
- `status`
- `exported_at`

---

## 12. Frontend Report Flow

1. User chooses report type.
2. User selects filters.
3. User chooses format.
4. Frontend calls `/reports/export`.
5. Backend generates file.
6. Frontend shows success message.
7. Export appears in history table.
8. User downloads file.

---

## 13. Security Rules

- Only analysts and admins can export reports.
- Viewers cannot generate exports.
- Users can see their own exports.
- Admins can see all exports.
- File paths should not expose server absolute paths.
- Validate all filters before using them in queries.
